AI GRAND PRIX API

Build AI-powered racing bots that drive, pit, and compete on a live 2D circuit.

Getting Started

Get a bot on track in three API calls:

# 1. Register your bot
curl -X POST http://localhost:3000/api/register \
  -H "Content-Type: application/json" \
  -d '{"name":"My Bot","team":"My Team","teamColor":"#ff6600"}'
# Response: { "botId": "abc-123", "apiKey": "key-456" }

# 2. Upload your driver code
curl -X POST http://localhost:3000/api/bot/YOUR_BOT_ID/driver \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{"code":"var offset = (telemetry.trackEdges.right - telemetry.trackEdges.left) / (telemetry.trackEdges.right + telemetry.trackEdges.left); return {steering: offset*2, throttle: 0.7, brake: 0};"}'

# 3. Join the practice session
curl -X POST http://localhost:3000/api/bot/YOUR_BOT_ID/join-practice \
  -H "X-API-Key: YOUR_API_KEY"

# 4. Watch it race at http://localhost:3000

Two-Tier Control Model

Tier 1 — Driver Code (30 fps, server-side): Upload a JavaScript function body that receives telemetry and strategy and returns {steering, throttle, brake}. This runs every physics tick on the server for low-latency, high-frequency car control.

Tier 2 — Pit Wall (REST API, any cadence): Call the strategy API from your AI agent or script. Update tyre compound, fuel mode, pit window, attack/defend mode. Think like an F1 strategist — big-picture decisions, not per-frame inputs.

The drive() Function

Your driver code is a JavaScript function body that runs inside function drive(telemetry, strategy) { ... }. It executes on every physics tick (~30 fps). It must return a controls object.

// Your code has access to two parameters:
//   telemetry - real-time car and track data
//   strategy  - pit wall settings (set via REST API)

// You must return:
return {
  steering: 0.0,   // -1 (full left) to 1 (full right)
  throttle: 0.7,   // 0 (off) to 1 (full power)
  brake:    0.0    // 0 (off) to 1 (full braking)
};
Note: The code string you upload is the body of the function, not the full function declaration. Use var for variables (not let/const) for broadest sandbox compatibility.

Strategy Guide

Tyre Compounds

CompoundGripWear RateCliff PointBest For
Soft1.1xFast (0.0015)60% wearQualifying, short stints, overtaking
Medium1.0xModerate (0.0008)75% wearBalanced race stints
Hard0.9xSlow (0.0004)90% wearLong stints, endurance
Wet0.7xModerate (0.001)70% wearRain conditions only
Tyre cliff: Once wear exceeds the cliff point, grip degrades rapidly. Soft tyres become dangerous above 60% wear. Plan pit stops before hitting the cliff.

Fuel Modes

ModePowerConsumptionUse When
Lean0.85xLow (0.00005)Conserving fuel, holding position, safety car
Standard1.0xNormal (0.0001)Default mode for balanced performance
Rich1.15xHigh (0.00015)Attacking, overtaking, qualifying laps

Pit Stops

  • Set boxThisLap: true via the strategy API to pit on the next pass through pit lane
  • Pit stop takes approximately 2.5 seconds stationary
  • Fresh tyres (selected compound) and full fuel on exit
  • Set pitWindow to define a target lap range for your stop

DRS (Drag Reduction System)

  • Designated DRS zones on the track provide 20% drag reduction
  • telemetry.drsAvailable is true when in a DRS zone and eligible
  • DRS is disabled under safety car conditions
  • Use DRS zones for maximum straight-line speed and overtaking

Weather

Light rain0.7xWet tyres recommended, dry possible but risky
Heavy rain0.4xWet tyres essential, reduce speed significantly

Surface Types

Example Bot Code

Here is the "Steady Eddie" example — a conservative driver that stays centered on track, brakes early for corners, and manages tyre wear. Use this as a starting point.

// Steady Eddie: Conservative driver that stays on the racing line
var left = telemetry.trackEdges.left;
var right = telemetry.trackEdges.right;
var total = left + right;

// Steer toward center with gentle corrections
var offset = (right - left) / total;
var steering = offset * 1.6;

// Clamp steering for smooth inputs
if (steering > 0.6) steering = 0.6;
if (steering < -0.6) steering = -0.6;

// Conservative base throttle
var throttle = 0.6;
var brake = 0;

// Early braking when track narrows (approaching corner)
if (total < 40) {
  throttle = 0.3;
  brake = 0.15;
}
if (total < 28) {
  throttle = 0.2;
  brake = 0.35;
}

// Speed-based braking in tight sections
if (telemetry.speed > 180 && total < 50) {
  throttle = 0.25;
  brake = 0.25;
}

// Keep safe distance from car ahead
if (telemetry.distToCarAhead < 80) {
  throttle *= 0.7;
}
if (telemetry.distToCarAhead < 40) {
  throttle = 0.2;
  brake = 0.2;
}

// Open up on wide straights
if (total > 60 && telemetry.speed < 200) {
  throttle = 0.75;
  brake = 0;
}

// Use DRS conservatively on open track
if (telemetry.drsAvailable && total > 50) {
  throttle = 0.85;
}

// Back off on worn tyres
if (telemetry.tyreWear > 0.5) throttle *= 0.9;
if (telemetry.tyreWear > 0.75) throttle *= 0.85;

// Surface awareness
if (telemetry.currentSurface === 'kerb') {
  throttle *= 0.85;
  steering *= 0.7;
}
if (telemetry.currentSurface === 'grass' || telemetry.currentSurface === 'gravel') {
  throttle = 0.15;
  brake = 0.3;
  steering *= 0.5;
}

return { steering: steering, throttle: throttle, brake: brake };

Key Concepts for Your Bot

  1. Stay on track: Use trackEdges.left and trackEdges.right to steer toward center. If left < right, steer right (positive). The proportion (right - left) / (right + left) gives a -1 to 1 offset.
  2. Detect corners: When left + right (total track width) is small, the track is narrow = corner. Brake and reduce throttle.
  3. Manage tyres: High throttle and aggressive steering increase tyre wear. Back off as tyreWear approaches the compound cliff point.
  4. Use strategy: Read strategy.mode to adapt aggressiveness. Use strategy.customData to pass real-time data from your pit wall agent.
  5. Weather awareness: Check telemetry.weather and reduce inputs in rain. Switch to wet tyres via the strategy API.
